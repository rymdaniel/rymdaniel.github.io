Vue.js: An Introduction

Why was Vue.js created? //PURPOSE

 Vue.js is a Javascript framework created by Evan You, a past developer at Google. He used to work a lot with Angular and figured he could just extract the really cool parts of it and make something far more lightweight without all the extra concepts. He said he was curious to see how Angular worked internal as well so he began Vue as an experiment and it slowly became Vue.js. he finished it in 2013 but it wasn't until 2014 when he publicly released it as an actual project.[5]

So what does it do?

 Vue.js allows one to create dynamic high-performance applications similar to react, but thanks to a virtual DOM it offers higher performance which is beneficial for complex apps. Essentially what Vue.js does is allows the content on a page to change quickly without having to reload the page.

Why use Vue.js versus other libraries and frameworks?

 Vue.js calls itself a progressive framework used for building user interfaces. It focuses on the view layer only but is powerful enough to power sophisticated single-page applications.

 Although it is similar to React with data binding and components, it focuses on approachability. This means that it makes sure that people who know the basics like HTML, CSS, and Javascript, can pick it up rather quickly. By being "lean" and having a "minimal core" it maintains flexibility as opposed to Ember or Angular who try to be opinionated on all the problems you may run into.[5] What this means is by being opinionated they force you to comply to a standard when different situations arise by guessing about the things you may run into. This forces you to be less flexible in your work.

 Vue takes a middle ground according to Evan You, by being minimal but still offering more complex things like routing and state management solutions. Another benefit is the optimization. Where a state change in React re-renders the entire component and child components, Vue keeps track of which components actually need to re-render, affecting only those.

How does Vue.js work? //WHAT I LEARNED

 To use Vue.js you can either use CDN by including it in the <span> tags within your HTML file. You can also install it by using Node Package Manager, installing Bower, or using the Vue CLI (Command Line Interface). You can think of Vue CLI like GIT if you are familiar with it. If you are using the Vue-CLI then your system needs to have Node.js and the npm command must be available on your system too.[6]

 The default HTML starts with a <div> with ID of "app" where all the content will be injected. A Javascript file called main.js initializes Vue by importing it and creating a new Vue instance. It takes an object as a parameter with three properties. EL: selects #app as the element where we will render everything into. TEMPLATE: Contains the HTML code used to generate output. COMPONENTS: Is the List of Vue.js components used in the template. The beauty of Vue.js is that it links the DOM and Script making them reactive (i.e. dynamic).

 Directives are prefixed with v- to indicate that they are special attributes provided by Vue. The following is an example of what using v- would be like.[3]

HTML

                    
                        <div id="app">
                            <span v-bind:title="message">
                                Hover over this to see something!
                            </span>
                        </div>

JAVASCRIPT

                    
                        var app = new Vue ({
                            el: "#app",
                            data: {
                                message: "You loaded this page on" + new Date().toLocaleString()
                            }
                        })

So what is this doing?

 v-bind:title is attaching the data's message as that span's title. When a user hovers over it it displays the date generated on DOM load. One of the benefits of Vue is that you don't have to do DOM manipulation yourself; they are handled by Vue.js. You can then also make a component accept a prop which is somewhat like an ID to make sure that only a specific component is updated and not every component that may share the same name.

Other examples that hit my soft spot

Project Helios

 I have a soft spot for the vapor the Vaporwave aesthetic and anything cyberpunk so this was a nice little find. Upon source file inspection, the webpack/static/js folder contains an app.js file with a Vue.js components and one for the glitch text that appears on the top left of the page. There is also a component for the music player which includes the progress bar, volume, and play/skip buttons.

 The things I like about the page are the simplicity and design choice but I can't help but feel I was expecting more. There is more to vaporwave than just the music at this point. There is imagery and abandonware that could fill the page with much more content for those wanting to look deeper into vaporwave. I get that the idea is to give people and introduction to vaporwave but it fells like nothing more than an excuse for the creator to possible share their favorite vaporwave racks to other people in a visual way (but who doesn't?). Don't get me wrong, it all fits together nicely, but I believe there should be more content aside from the text converter and the news letter.

Hacker News Clone

 I finally decided to check out what Hacker News was a few days ago and plan on visiting it frequently to keep up with the latest tech news. I then happened to find this hacker news clone made with Vue. It loads fake articles, simulating real-time posting without reloading the page, so I know Vue and all the other back-end stuff is working. Looking at the source code you can see data, computed, props, and mounted properties which are all part of Vue.js and how it implements things onto the page.

 The only thing I dislike about it is the fact that although it's decided to show an image preview of the article, which may be good in some cases to get an idea what the article is about, they aren't anything more than screenshots of the article itself. If the header is interesting enough then chances are a viewer will check it out, regardless if it has an image or not. The idea of a preview is to support the title but a screenshot of the article itself does nothing other than take up space on the page when it could simply be a list view like hackernews already does.
 (And yes, I'm totally aware I'm doing exactly what I'm critiquing).

Final thoughts... //CONCLUSION

Overall impression

 Without knowing too much about Vue.js it's clear that its very powerful and fast as its creator, Evan You, intended. Given the industry today I feel it's necessary to either learn Angular, React, and maybe even both with focus on one of them. But with the power that Vue.js has it seems the only reason it doesn't pick up or have as much market share as Angular and React do is due to a lack of an established corporate sponsor. I plan on learning it not only because its well made and faster at times than the other two most popular frameworks, but because it seems to be the best entry level framework for those who have HTML, CSS, and JS knowledge already compared to the learning curve of Angular for example.

Are there benefits to learning all this?

 It's a really good tool to make your page reactive and fast without having to worry too much about coding everything by hand and leaving some the heavy lifting to the framework. Although it is a good thing to know how to do something in vanilla JS and not depend too much on someone or something else to do things for you, once you learn those things on your own it seems reasonable to move onto libraries and frameworks.

 Why? Because now that you know things work manually, you can better grasp how to do things with these tools at hand make your coding experience more efficient and clean.
